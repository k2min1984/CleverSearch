## 시나리오 C — 상세 필터 (오늘 수정 완료)

### 수정 전 문제

- `include_keywords: ["울산지방청"]` → 본문에 없고 파일명에만 있어서 결과 0건

### 수정 후 동작

- 필터가 `all_text` OR `Title` 양쪽을 모두 확인
- 파일명이 Title로 저장되므로 울산지방청이 Title에서 매칭됨

### 검증 순서

```
POST /api/v1/search/query
{
  "query": "연구개발계획서",
  "include_keywords": ["울산지방청"],
  "exclude_keywords": ["작성중"],
  "file_ext": "pdf"
}
→ 기대: 1. 연구개발계획서 양식_울산지방청.pdf 1건 반환
```

### 화면 멘트

> "상세 검색 include/exclude 필터가 본문뿐 아니라 제목(파일명)까지 검색하도록 오늘 수정했습니다.  
> '울산지방청'은 본문엔 없고 파일명에만 있는데 이제 정상 매칭됩니다.  
> '작성중' exclude 적용 시 연구개발계획서\_작성중.docx는 제외됩니다."

## 시나리오 G — 실패 검색 추적 (오늘 수정 완료)

### 수정 전 문제

- `ZXCVBNM존재하지않는키워드` 검색 시 벡터 유사도로 결과가 반환됨
- `is_failed=False`로 기록되어 실패 검색어 집계 누락

### 수정 후 동작

- 하이브리드 검색 후 **제목/본문 정확 구문 카운트 쿼리** 별도 실행
- 정확 구문 결과 = 0이면 `is_failed=True` 로 기록 (벡터 결과 무관)

### 검증 순서

```
POST /api/v1/search/query { "query": "ZXCVBNM존재하지않는키워드" }
→ 검색결과는 벡터 유사도로 나올 수 있음
→ GET /api/v1/admin/failed-keywords 에서 집계 확인
```

### 화면 멘트

> "실패 검색 판정이 벡터 단독 매칭이 아닌 렉시컬 검색 기준으로 수정됐습니다.  
> 관리자 화면에서 어떤 키워드가 실패했는지 추적 가능하고,  
> 이를 통해 서비스에 부족한 문서나 동의어 사전을 보완할 수 있습니다."

## 보안/폐쇄망/운영 준비도 요약 (회의 안건)

### ✅ 폐쇄망 동작 가능

- 외부 인터넷 호출 코드 없음
- AI 임베딩 모델은 최초 1회 다운로드 후 로컬 캐시 사용 (`~/.cache/huggingface`)
- **배포 시 필요 조치**: pip wheel 오프라인 패키지 / 모델 캐시 미리 복사

### ✅ 오늘 적용된 보안 패치

| 항목                      | 수정 내용                                     |
| ------------------------- | --------------------------------------------- |
| 파일 업로드 크기 제한     | 100MB 초과 시 413 반환 (서버 DoS 방지)        |
| include/exclude 필터      | Title + all_text 양쪽 검색 (누락 방지)        |
| 실패 검색 판정            | 렉시컬 카운트 기준으로 재정의                 |
| 이미지형 PDF OCR          | 텍스트 없는 PDF → PyMuPDF 렌더링 후 Tesseract |
| 검색어 최대 길이          | 200자 제한 (Pydantic 스키마)                  |
| include/exclude 최대 개수 | 각 10개 제한                                  |

### ❌ 추가 보완 필요 (대국민 서비스 전)

| 우선순위 | 항목                     | 설명                                                   |
| -------- | ------------------------ | ------------------------------------------------------ |
| HIGH     | 인증/인가 없음           | 모든 API 누구나 접근 가능. JWT 또는 세션 필요          |
| HIGH     | 관리자 삭제 API 무방비   | `/clear-all-data`, `/delete-index` 인증 없이 호출 가능 |
| HIGH     | Rate Limiting 없음       | 검색/업로드 무제한 호출 가능 → DDoS 취약               |
| MED      | CORS 전체 개방           | `allow_origins=["*"]` → 서비스 도메인으로 제한 필요    |
| MED      | SSL 인증서 검증 우회     | `verify_certs=False` → 운영 시 유효한 인증서 사용      |
| MED      | 비밀번호 기본값 하드코딩 | `Admin123!` → 운영 환경변수로 반드시 교체              |
| LOW      | Docker 포트 외부 노출    | 9200, 5432 포트 → expose로 변경해 내부망만 허용        |

## 화면 순서 요약 (데모 진행 흐름)

```
[1] https://localhost:8000       → 메인 화면 오픈 (5초)
[2] 파일 업로드 탭               → 9개 파일 드래그 업로드, 결과 확인
[3] test.* 파일 재업로드         → skipped 확인 (중복 차단 동작)
[4] 검색창 "산업기술혁신사업"     → 상위 결과 確認, 스니펫 하이라이트
[5] 상세 검색 "연구개발계획서"    → include:울산지방청 / exclude:작성중 / 확장자:pdf
[6] 검색창 "연구개발계확서" (오타) → 교정 후 동일 결과
[7] 자동완성 입력                → prefix 추천 목록
[8] "ZXCVBNM존재하지않는키워드"   → 결과 확인 후 관리자에서 실패 확인
[9] 관리자 탭                   → 인기/실패/색인/로그 확인
[10] 보완점 슬라이드             → 보안 요약 공유
```
