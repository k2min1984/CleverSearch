# CleverSearch 회의기반 실행일정 (2026-04-22)

## 1) 회의 결론 요약

- 다음 주 핵심 태스크는 3개로 고정한다.
  - SMB 연동
  - 스케줄러 안정화
  - 에이전트 구조 확정
- 업로드 기능과 색인 기능은 개념적으로 분리한다.
- 관리자 업로드 UI의 고급 비동기 팝업(드래그 이동/최소화/다중 상태창)은 현 시점 우선순위에서 제외한다.
- 폼 전송 기반의 안정 동작을 우선하고, 백그라운드 처리/에이전트 방식으로 운영한다.
- 데모 대비로 UI/UX 정비와 해상도 대응(데스크톱 중심, 태블릿 최소 대응)을 병행한다.

## 2) 범위/제외

### 이번 주기 범위 (In Scope)

- SMB 접속 기반 파일 감지/처리 경로 확정
- 스케줄러와 DB 연계 안정화
- 에이전트 동작 정책 확정(파일 감지, 기준 시점, 실패 복구)
- WAS 환경 통합 점검(현재 소스 머지 후 실행 검증)
- 데모용 UI/UX 정비(레이아웃/정렬/가독성/차트 구성)
- 요약 PPT용 스펙/소개 자료 정리

### 이번 주기 제외 (Out of Scope)

- 대용량 업로드 고급 UX(플로팅 상태창, 페이지 간 지속 미니 패널)
- 청크 업로드/재개/취소 고도화 전체 구현
- 완전 모바일(휴대폰) 최적화

## 4) 우선순위 (중요도 순)

1. SMB + 스케줄 + 에이전트(핵심 구조)
2. WAS 통합 실행 안정화
3. UI/UX 데모 품질
4. PPT 스펙/소개 자료

## 5) 리스크 및 대응

- STT 정확도 90% 요구
  - 대응: 비용/정확도/온프레미스 제약을 분리해 비교표 작성 후 의사결정
- SMB 테스트 환경 미비
  - 대응: 테스트용 서버/공유경로 먼저 확보, 로컬 가정 개발 지양
- 업로드/색인 혼재로 인한 구조 복잡도
  - 대응: 기능 경계(책임 분리) 문서화 후 단계별 분리
- UI 고도화 과투입
  - 대응: 데모 품질 기준(깨짐 없음, 가독성 확보)까지로 범위 제한
