## 시나리오 1. 관리자 로그인/로그아웃 〈일정 #39〉

### 목적

관리자 페이지 접근 시 로그인이 필수인지, 로그아웃 후 콘텐츠가 숨겨지는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                      | 예상 결과                                                                                                          |
| :--: | ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
|  1   | 브라우저에서 **https://localhost:8443/admin** 을 엽니다                             | "관리자 로그인" 화면만 보입니다. 탭이나 내부 콘텐츠는 보이지 않습니다                                              |
|  2   | 아이디에 `admin`, 비밀번호에 `admin123!` 을 입력하고 **[로그인]** 버튼을 클릭합니다 | 상단에 14개 탭(파일 업로드, 문서 목록, …, 검색 품질, 초기화)이 나타나고, 오른쪽 위에 `admin` 사용자명이 표시됩니다 |
|  3   | 오른쪽 위의 **[로그아웃]** 버튼을 클릭합니다                                        | 다시 로그인 화면으로 돌아갑니다. 탭과 내부 콘텐츠가 모두 사라집니다                                                |
|  4   | 아이디에 `admin`, 비밀번호에 `틀린비번` 을 입력하고 **[로그인]** 을 클릭합니다      | "로그인 실패" 에러 메시지가 표시됩니다. 로그인되지 않습니다                                                        |

**✅ 합격 기준:** 4단계 모두 예상대로 동작

## 시나리오 2. 전체 초기화 〈일정 #8〉

### 목적

테스트 시작 전 깨끗한 상태를 만들기 위해 OpenSearch와 DB를 동시에 초기화

### 실행 방법

| 순서 | 화면에서 할 일                                               | 예상 결과                                                                 |
| :--: | ------------------------------------------------------------ | ------------------------------------------------------------------------- |
|  1   | 관리자 페이지에서 `admin` / `admin123!` 로 로그인합니다      | 로그인 성공                                                               |
|  2   | 맨 오른쪽 **[초기화]** 탭을 클릭합니다                       | "전체 데이터 초기화"와 "인덱스 완전 삭제 + DB 초기화" 버튼 2개가 보입니다 |
|  3   | **[전체 데이터 초기화 (OpenSearch + DB)]** 버튼을 클릭합니다 | "전체 초기화 완료 (OpenSearch + DB)" 메시지가 나타납니다                  |
|  4   | **[문서 목록]** 탭을 클릭하고 **[새로고침]** 을 누릅니다     | 문서가 0건으로 표시됩니다                                                 |
|  5   | **[검색 로그]** 탭을 클릭하고 **[새로고침]** 을 누릅니다     | 검색 로그가 0건으로 표시됩니다                                            |
|  6   | **[최근 검색어]** 탭을 클릭하고 **[새로고침]** 을 누릅니다   | 최근 검색어가 0건으로 표시됩니다                                          |

**✅ 합격 기준:** 문서·검색로그·최근검색어 모두 0건

## 시나리오 3. 파일 업로드 〈일정 #8~#14, #47~#49〉

### 목적

다양한 확장자의 파일을 업로드하고, 중복 파일은 자동 차단되는지 확인 (PDF/HWP/HWPX/Office 텍스트 추출, 제어문자 정제, Hash 중복 방지, 더미 데이터 검증 포함)

### 실행 방법

| 순서 | 화면에서 할 일                                                                               | 예상 결과                                                                        |
| :--: | -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
|  1   | **[파일 업로드]** 탭을 클릭합니다                                                            | 파일 드래그&드롭 영역이 나타납니다                                               |
|  2   | 준비한 5개 파일을 드래그하여 업로드 영역에 놓습니다 (또는 영역을 클릭하여 파일을 선택합니다) | 각 파일마다 업로드 진행 표시 후 결과가 나타납니다                                |
|  3   | 각 파일의 상태를 확인합니다                                                                  | 5개 모두 **status: success** 가 표시됩니다                                       |
|  4   | **[문서 목록]** 탭 → **[새로고침]** 을 클릭합니다                                            | 총 **5건**이 표시됩니다. 파일명·확장자·카테고리를 확인합니다                     |
|  5   | 다시 **[파일 업로드]** 탭으로 이동합니다                                                     |                                                                                  |
|  6   | 아까 업로드한 `연구개발계획서_작성중.docx` 파일을 **다시 한 번** 업로드합니다                | **status: skipped**, 메시지 "내용 중복: 연구개발계획서작성중.docx" 가 표시됩니다 |

**✅ 합격 기준:** 5개 파일 success + 중복 파일 skipped

## 시나리오 4. 기본 검색 〈일정 #24, #25, #29〉

### 목적

업로드한 문서를 다양한 키워드로 검색하여 관련 문서가 정확히 나오는지 확인 (하이브리드 검색 엔진, 검색어 전처리, 항목별 가중치 부여 포함)

### 실행 방법

| 순서 | 화면에서 할 일                                                 | 예상 결과                                                                                                        |
| :--: | -------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
|  1   | 브라우저에서 **https://localhost:8443/** 을 엽니다             | 검색창과 "인기 검색어" 태그가 보이는 메인 화면이 나타납니다                                                      |
|  2   | 검색창에 `연구개발계획서` 를 입력하고 **[검색]** 을 클릭합니다 | **3건** 결과가 검색됩니다. 상위에 `연구개발계획서 양식_울산지방청.pdf`, `연구개발계획서_작성중.docx` 가 보입니다 |
|  3   | 검색창을 지우고 `AI개발진행` 을 입력하여 검색합니다            | **2건** 결과. `AI개발진행-20251201.pptx` 가 상위에 나타납니다                                                    |
|  4   | `통합검색솔루션` 으로 검색합니다                               | **3건** 결과. `통합검색솔루션_종합문서.xlsx` 가 상위에 나타납니다                                                |
|  5   | `운영지침` 으로 검색합니다                                     | **5건** 결과. `2026_운영지침_보안.pdf.jpg` 가 상위에 나타납니다                                                  |
|  6   | `보안` 으로 검색합니다                                         | **4건** 결과. `2026_운영지침_보안.pdf.jpg` 가 상위에 나타납니다                                                  |

**✅ 합격 기준:** 각 키워드에 해당하는 파일이 결과 상위에 노출

## 시나리오 5. 상세 검색 〈일정 #26, #27, #28〉

### 목적

포함 키워드, 제외 키워드, 파일 형식, 기간 필터가 정상 작동하는지 확인 (상세검색 포함/제외, 결과 내 재검색, 날짜/기간 범위 검색 포함)

### 실행 방법

| 순서 | 화면에서 할 일                                                                                            | 예상 결과                                                     |
| :--: | --------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------- |
|  1   | 메인 화면에서 `연구개발계획서` 로 검색합니다                                                              | **3건** 결과                                                  |
|  2   | 검색 결과 화면의 필터 영역에서 **"포함 키워드"** 입력칸에 `울산지방청` 을 입력하고 다시 검색합니다        | **1건** — 울산지방청.pdf만 남습니다                           |
|  3   | 포함 키워드를 지우고, **"제외 키워드"** 입력칸에 `작성중` 을 입력하여 `연구개발계획서` 를 다시 검색합니다 | **2건** — `연구개발계획서_작성중.docx` 가 결과에서 제외됩니다 |
|  4   | 제외 키워드를 지우고, 검색 결과 화면에서 **파일 형식** 드롭다운을 `PDF 문서` 로 변경합니다                | PDF 파일만 남습니다 (**1건** — 울산지방청.pdf)                |
|  5   | **기간** 드롭다운을 `최근 24시간` 으로 변경합니다                                                         | 오늘 업로드한 문서만 표시됩니다                               |

**✅ 합격 기준:** 각 필터별로 결과가 정확히 좁혀짐

## 시나리오 6. 오타 교정 검색 〈일정 #17, #31〉

### 목적

오타가 포함된 검색어를 입력해도 자동으로 교정하여 결과를 보여주는지 확인 (hanspell 기반 한글 오타 자동 교정)

### 실행 방법

| 순서 | 화면에서 할 일                                                         | 예상 결과                                                                                               |
| :--: | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
|  1   | 메인 화면 검색창에 `계휙서` (오타)를 입력하고 **[검색]** 을 클릭합니다 | 상단에 `입력어 계휙서 를 계획서 로 교정하여 검색했습니다.` 메시지가 표시되고, **2건** 결과가 나타납니다 |
|  2   | `산엽` (오타)으로 검색합니다                                           | `산엽` → `산업` 으로 교정되어 검색됩니다. 교정 메시지가 표시되고 결과가 나타납니다                      |

**✅ 합격 기준:** 오타 입력 시 교정 메시지 + 정상 검색 결과

## 시나리오 7. 초성 검색 〈일정 #35〉

### 목적

한글 초성(ㄱ, ㄴ, ㄷ…)만 입력해도 해당 초성 패턴을 포함하는 문서가 검색되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                                  | 예상 결과                                                                                                             |
| :--: | ----------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
|  1   | 메인 화면 검색창에 `ㅇㄱㄱㅂㄱㅎㅅ` (연구개발계획서의 초성)를 입력하고 **[검색]** 을 클릭합니다 | **1건** 결과가 나타납니다. `연구개발계획서 양식_울산지방청.pdf` 가 표시됩니다                                         |
|  2   | `ㅂㅇ` (보안의 초성)으로 검색합니다                                                             | **4건** 결과. `통합검색솔루션_종합문서.xlsx`, `연구개발계획서_작성중.docx` 등이 검색됩니다                            |
|  3   | `ㅌㅎㄱㅅ` (통합검색의 초성)으로 검색합니다                                                     | **0건** 결과. 단축 초성은 매칭되지 않을 수 있습니다                                                                   |
|  4   | 검색창에 `ㅇㄱ` 를 천천히 입력합니다 (검색 버튼 누르지 않음)                                    | 자동완성 드롭다운에 `연구개발계획서_작성중.docx`, `연구개발계획서 양식_울산지방청.pdf` 등 초성 매칭 후보가 표시됩니다 |

**✅ 합격 기준:** 초성만으로 정확한 문서 검색 + 초성 자동완성 후보 표시

### 실행 방법

| :--: | ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |

## 시나리오 10. 문서 개별 삭제 〈일정 #8〉

### 목적

특정 문서를 삭제하면 OpenSearch와 DB에서 모두 제거되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                     | 예상 결과                                                                   |
| :--: | ------------------------------------------------------------------ | --------------------------------------------------------------------------- |
|  1   | 관리자 페이지 → **[문서 목록]** 탭 → **[새로고침]**                | 현재 문서 건수를 확인합니다 (**5건**)                                       |
|  2   | `통합검색솔루션_종합문서.xlsx` 문서의 **[삭제]** 버튼을 클릭합니다 | 확인 팝업 → "삭제 완료" 메시지                                              |
|  3   | **[새로고침]** 을 누릅니다                                         | 총 건수가 **4건**으로 줄어있습니다                                          |
|  4   | 메인 화면에서 `통합검색솔루션` 으로 검색합니다                     | 삭제 전 3건이던 결과가 **2건**으로 줄어듭니다 (xlsx 문서가 결과에서 사라짐) |
