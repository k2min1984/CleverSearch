## 시나리오 1. 관리자 로그인/로그아웃

### 목적

관리자 페이지 접근 시 로그인이 필수인지, 로그아웃 후 콘텐츠가 숨겨지는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                      | 예상 결과                                                                                               |
| :--: | ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
|  1   | 브라우저에서 **https://localhost:8443/admin** 을 엽니다                             | "관리자 로그인" 화면만 보입니다. 탭이나 내부 콘텐츠는 보이지 않습니다                                   |
|  2   | 아이디에 `admin`, 비밀번호에 `admin123!` 을 입력하고 **[로그인]** 버튼을 클릭합니다 | 상단에 13개 탭(파일 업로드, 문서 목록, …, 초기화)이 나타나고, 오른쪽 위에 `admin` 사용자명이 표시됩니다 |
|  3   | 오른쪽 위의 **[로그아웃]** 버튼을 클릭합니다                                        | 다시 로그인 화면으로 돌아갑니다. 탭과 내부 콘텐츠가 모두 사라집니다                                     |
|  4   | 아이디에 `admin`, 비밀번호에 `틀린비번` 을 입력하고 **[로그인]** 을 클릭합니다      | "로그인 실패" 에러 메시지가 표시됩니다. 로그인되지 않습니다                                             |

**✅ 합격 기준:** 4단계 모두 예상대로 동작

---

## 시나리오 2. 전체 초기화

### 목적

테스트 시작 전 깨끗한 상태를 만들기 위해 OpenSearch와 DB를 동시에 초기화

### 실행 방법

| 순서 | 화면에서 할 일                                               | 예상 결과                                                                 |
| :--: | ------------------------------------------------------------ | ------------------------------------------------------------------------- |
|  1   | 관리자 페이지에서 `admin` / `admin123!` 로 로그인합니다      | 로그인 성공                                                               |
|  2   | 맨 오른쪽 **[초기화]** 탭을 클릭합니다                       | "전체 데이터 초기화"와 "인덱스 완전 삭제 + DB 초기화" 버튼 2개가 보입니다 |
|  3   | **[전체 데이터 초기화 (OpenSearch + DB)]** 버튼을 클릭합니다 | "전체 초기화 완료 (OpenSearch + DB)" 메시지가 나타납니다                  |
|  4   | **[문서 목록]** 탭을 클릭하고 **[새로고침]** 을 누릅니다     | 문서가 0건으로 표시됩니다                                                 |
|  5   | **[검색 로그]** 탭을 클릭하고 **[새로고침]** 을 누릅니다     | 검색 로그가 0건으로 표시됩니다                                            |
|  6   | **[최근 검색어]** 탭을 클릭하고 **[새로고침]** 을 누릅니다   | 최근 검색어가 0건으로 표시됩니다                                          |

**✅ 합격 기준:** 문서·검색로그·최근검색어 모두 0건

## 시나리오 3. 파일 업로드

### 목적

다양한 확장자의 파일을 업로드하고, 중복 파일은 자동 차단되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                               | 예상 결과                                                                        |
| :--: | -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
|  1   | **[파일 업로드]** 탭을 클릭합니다                                                            | 파일 드래그&드롭 영역이 나타납니다                                               |
|  2   | 준비한 5개 파일을 드래그하여 업로드 영역에 놓습니다 (또는 영역을 클릭하여 파일을 선택합니다) | 각 파일마다 업로드 진행 표시 후 결과가 나타납니다                                |
|  3   | 각 파일의 상태를 확인합니다                                                                  | 5개 모두 **status: success** 가 표시됩니다                                       |
|  4   | **[문서 목록]** 탭 → **[새로고침]** 을 클릭합니다                                            | 총 **5건**이 표시됩니다. 파일명·확장자·카테고리를 확인합니다                     |
|  5   | 다시 **[파일 업로드]** 탭으로 이동합니다                                                     |                                                                                  |
|  6   | 아까 업로드한 `연구개발계획서_작성중.docx` 파일을 **다시 한 번** 업로드합니다                | **status: skipped**, 메시지 "내용 중복: 연구개발계획서작성중.docx" 가 표시됩니다 |

**✅ 합격 기준:** 5개 파일 success + 중복 파일 skipped

---

## 시나리오 4. 기본 검색

### 목적

업로드한 문서를 다양한 키워드로 검색하여 관련 문서가 정확히 나오는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                 | 예상 결과                                                                                                             |
| :--: | -------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
|  1   | 브라우저에서 **https://localhost:8443/** 을 엽니다             | 검색창과 "인기 검색어" 태그가 보이는 메인 화면이 나타납니다                                                           |
|  2   | 검색창에 `연구개발계획서` 를 입력하고 **[검색]** 을 클릭합니다 | **약 4개**의 결과가 검색됩니다. 상위에 `연구개발계획서_작성중.docx`, `연구개발계획서 양식_울산지방청.pdf` 가 보입니다 |
|  3   | 검색창을 지우고 `AI개발진행` 을 입력하여 검색합니다            | **약 2개** 결과. `AI개발진행-20251201.pptx` 가 상위에 나타납니다                                                      |
|  4   | `통합검색솔루션` 으로 검색합니다                               | **약 2개** 결과. `통합검색솔루션_종합문서.xlsx` 가 상위에 나타납니다                                                  |
|  5   | `운영지침` 으로 검색합니다                                     | **약 5개** 결과. `2026_운영지침_보안.pdf.jpg` 가 상위에 나타납니다                                                    |
|  6   | `보안` 으로 검색합니다                                         | **약 3개** 결과. `2026_운영지침_보안.pdf.jpg` 가 상위에 나타납니다                                                    |

**✅ 합격 기준:** 각 키워드에 해당하는 파일이 결과 상위에 노출

## 시나리오 5. 상세 검색 (필터)

### 목적

포함 키워드, 제외 키워드, 파일 형식, 기간 필터가 정상 작동하는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                                                  | 예상 결과                                                    |
| :--: | --------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
|  1   | 메인 화면에서 `연구개발계획서` 로 검색합니다                                                                    | 약 4개 결과                                                  |
|  2   | 검색 결과 화면의 필터 영역에서 검색창에 `연구개발계획서 울산지방청` 처럼 포함 키워드를 추가하여 다시 검색합니다 | **1건** — 울산지방청 관련 문서만 남습니다                    |
|  3   | `연구개발계획서` 로 검색 후, 상세검색에서 제외조건에 `작성중` 을 입력하여 검색합니다                            | `연구개발계획서_작성중.docx` 가 결과에서 제외됩니다 (약 3건) |
|  4   | 검색 결과 화면에서 **파일 형식** 드롭다운을 `PDF 문서` 로 변경합니다                                            | PDF 파일만 남습니다 (**1건** — 울산지방청.pdf)               |
|  5   | **기간** 드롭다운을 `최근 24시간` 으로 변경합니다                                                               | 오늘 업로드한 문서만 표시됩니다                              |

**✅ 합격 기준:** 각 필터별로 결과가 정확히 좁혀짐

## 시나리오 6. 오타 교정 검색

### 목적

오타가 포함된 검색어를 입력해도 자동으로 교정하여 결과를 보여주는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                         | 예상 결과                                                                                                          |
| :--: | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
|  1   | 메인 화면 검색창에 `계휙서` (오타)를 입력하고 **[검색]** 을 클릭합니다 | 상단에 `입력어 계휙서 를 연구개발계획서 로 교정하여 검색했습니다.` 메시지가 표시되고, **약 2개** 결과가 나타납니다 |
|  2   | `산엽` (오타)으로 검색합니다                                           | 교정 메시지가 표시되고 **약 3개** 결과가 나타납니다                                                                |

**✅ 합격 기준:** 오타 입력 시 교정 메시지 + 정상 검색 결과

## 시나리오 7. 자동완성

### 목적

검색창에 글자를 입력할 때 자동완성 후보가 드롭다운으로 나타나는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                           | 예상 결과                                                                                                                                      |
| :--: | ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
|  1   | 메인 화면 검색창에 `연구` 를 천천히 입력합니다 (검색 버튼은 누르지 않음) | 검색창 아래에 자동완성 드롭다운이 나타납니다. `연구개발계획서 양식_울산지방청.pdf`, `연구개발계획서_작성중.docx` 등 문서명이 후보로 표시됩니다 |
|  2   | 드롭다운에서 원하는 항목을 클릭합니다                                    | 해당 키워드로 검색이 실행됩니다                                                                                                                |

**✅ 합격 기준:** 자동완성 후보가 2개 이상 표시

## 시나리오 8. 인기검색어 / 최근검색 / 실패검색

### 목적

검색 이력이 올바르게 기록되고 관리(삭제)할 수 있는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                               | 예상 결과                                                      |
| :--: | -------------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
|  1   | 메인 화면에서 `연구개발계획서`, `AI개발진행`, `보안`, `운영지침` 등 여러 키워드로 검색합니다 | 각 검색이 정상 수행됩니다                                      |
|  2   | 메인 화면 상단의 "인기 검색어" 태그를 확인합니다                                             | 방금 검색한 키워드가 태그로 표시됩니다                         |
|  3   | 관리자 페이지 → **[인기검색]** 탭 → **[새로고침]**                                           | 검색 횟수가 많은 순서대로 키워드 목록이 표시됩니다             |
|  4   | 관리자 페이지 → **[최근 검색어]** 탭 → **[새로고침]**                                        | 가장 최근에 검색한 키워드가 최상단에 표시됩니다                |
|  5   | 관리자 페이지 → **[검색 로그]** 탭 → **[새로고침]**                                          | 모든 검색 이력이 시간·키워드·결과수·성공여부와 함께 표시됩니다 |
|  6   | 메인 화면으로 돌아가서 `ZXCVBNM존재하지않는키워드` 로 검색합니다                             | "일치하는 문서를 찾지 못했습니다." (total=0)                   |
|  7   | 관리자 페이지 → **[실패 검색어]** 탭 → **[새로고침]**                                        | 방금 검색한 키워드가 실패 목록에 표시됩니다                    |

**✅ 합격 기준:** 인기/최근/검색로그/실패 검색어가 모두 올바르게 기록됨

## 시나리오 9. 최근 검색어 삭제

### 목적

최근 검색어를 개별 삭제 또는 전체 삭제할 수 있는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                | 예상 결과                   |
| :--: | ------------------------------------------------------------- | --------------------------- |
|  1   | 관리자 페이지 → **[최근 검색어]** 탭 → **[새로고침]**         | 최근 검색 목록이 표시됩니다 |
|  2   | 목록에서 아무 항목의 **[삭제]** 버튼을 클릭합니다             | 해당 항목만 사라집니다      |
|  3   | **[새로고침]** → 목록에서 삭제된 항목이 없어졌는지 확인합니다 | 삭제 확인 완료              |

**✅ 합격 기준:** 단건 삭제 정상 동작

---

## 시나리오 10. 문서 개별 삭제

### 목적

특정 문서를 삭제하면 OpenSearch와 DB에서 모두 제거되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                      | 예상 결과                                   |
| :--: | --------------------------------------------------- | ------------------------------------------- |
|  1   | 관리자 페이지 → **[문서 목록]** 탭 → **[새로고침]** | 현재 문서 건수를 확인합니다 (예: 5건)       |
|  2   | 삭제할 문서의 **[삭제]** 버튼을 클릭합니다          | 확인 팝업 → "삭제 완료" 메시지              |
|  3   | **[새로고침]** 을 누릅니다                          | 총 건수가 1건 줄어있습니다 (예: 4건)        |
|  4   | 메인 화면에서 삭제한 문서명으로 검색합니다          | 삭제한 문서가 검색 결과에 나타나지 않습니다 |

**✅ 합격 기준:** 삭제 후 문서 수 감소 + 검색 결과에서도 사라짐

---

## 시나리오 11. 대시보드

### 목적

대시보드에 검색 통계가 올바르게 표시되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                 | 예상 결과                                                                     |
| :--: | ---------------------------------------------- | ----------------------------------------------------------------------------- |
|  1   | 관리자 페이지 → **[대시보드]** 탭을 클릭합니다 | 4개의 카드가 보입니다: **전체 로그**, **실패 로그**, **실패율**, **스케줄러** |
|  2   | **전체 로그** 숫자를 확인합니다                | 지금까지 실행한 검색 횟수 만큼 숫자가 표시됩니다 (예: 21건 이상)              |
|  3   | 아래 **트렌드 차트** 를 확인합니다             | 오늘 날짜에 성공/실패 막대 그래프가 표시됩니다                                |
|  4   | **인기 키워드 차트** (도넛)를 확인합니다       | 검색량이 많은 키워드 순서로 도넛 차트가 표시됩니다                            |
|  5   | **[새로고침]** 을 클릭합니다                   | 최신 데이터로 갱신됩니다                                                      |

**✅ 합격 기준:** 4개 카드 + 트렌드 차트 + 키워드 차트 모두 정상 표시

---

## 시나리오 12. 사전 관리 (동의어/불용어/사용자사전)

### 목적

검색 사전에 단어를 등록·조회할 수 있는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                                                                       | 예상 결과                                                         |
| :--: | ------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------- |
|  1   | 관리자 페이지 → **[사전 관리]** 탭을 클릭합니다                                                                                      | 사전 유형 드롭다운, term/replacement 입력칸, 저장 버튼이 보입니다 |
|  2   | **사전 유형** 을 `synonym` 으로 선택하고, **term** 에 `AI`, **replacement** 에 `인공지능` 을 입력한 뒤 **[단건 저장]** 을 클릭합니다 | 저장 성공 메시지 (term: AI, replacement: 인공지능)                |
|  3   | **사전 유형** 을 `user` 로 선택하고, **term** 에 `계휙서`, **replacement** 에 `계획서` 를 입력한 뒤 **[단건 저장]** 을 클릭합니다    | 저장 성공                                                         |
|  4   | **사전 유형** 을 `stopword` 로 선택하고, **term** 에 `에` 를 입력한 뒤 **[단건 저장]** 을 클릭합니다                                 | 저장 성공                                                         |
|  5   | **[목록 조회]** 를 클릭합니다                                                                                                        | 등록한 사전 항목들이 목록에 표시됩니다                            |

**✅ 합격 기준:** 3종류 사전 모두 등록 성공 + 목록에서 확인 가능

### 목적

외부 데이터 소스(SMB 공유 폴더, 외부 DB)를 등록·조회할 수 있는지 확인

> ⚠️ **참고:** 실제 SMB 경로나 DB 연결이 준비되어 있어야 동기화 테스트가 가능합니다

## 시나리오 14. 스케줄러

### 목적

자동 색인 스케줄러의 시작/중지/상태 조회가 정상 동작하는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                  | 예상 결과                                               |
| :--: | ----------------------------------------------- | ------------------------------------------------------- |
|  1   | 관리자 페이지 → **[스케줄러]** 탭 클릭          | 간격(초) 입력칸, 시작/중지/상태조회 버튼이 보입니다     |
|  2   | **[상태 조회]** 를 클릭합니다                   | `running: false`, `interval_seconds: 120` 이 표시됩니다 |
|  3   | 간격을 `120` 으로 두고 **[시작]** 을 클릭합니다 | `running: true` 로 변경됩니다                           |
|  4   | **[중지]** 를 클릭합니다                        | `running: false` 로 돌아갑니다                          |

**✅ 합격 기준:** 시작→running:true, 중지→running:false

## 시나리오 16. 권한 분리 (RBAC)

### 목적

`viewer` 계정은 조회만 가능하고, 관리 작업(사전 등록, 스케줄러 시작 등)은 차단되는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                                                                    | 예상 결과                                      |
| :--: | --------------------------------------------------------------------------------- | ---------------------------------------------- |
|  1   | 관리자 페이지에서 **[로그아웃]** 합니다                                           | 로그인 화면으로 돌아갑니다                     |
|  2   | 아이디 `viewer`, 비밀번호 `viewer123!` 로 로그인합니다                            | 로그인 성공. 사용자명에 `viewer` 표시          |
|  3   | **[문서 목록]** 탭 → **[새로고침]** 을 클릭합니다                                 | 정상 조회됩니다 (viewer는 조회 권한 있음)      |
|  4   | **[사전 관리]** 탭 → synonym / AI / 인공지능 입력 → **[단건 저장]** 을 클릭합니다 | **"권한 없음" (403 에러)** 메시지가 표시됩니다 |
|  5   | **[스케줄러]** 탭 → **[시작]** 을 클릭합니다                                      | **"권한 없음" (403 에러)** 메시지가 표시됩니다 |
|  6   | **[로그아웃]** 후 `admin` 으로 다시 로그인합니다                                  | admin은 모든 기능이 정상 동작합니다            |

**✅ 합격 기준:** viewer는 조회 OK, 등록/수정/삭제/시작 시 403 차단

## 시나리오 17. 이름 검색 정밀도

### 목적

문서에 없는 사람 이름으로 검색했을 때 엉뚱한 결과가 나오지 않는지 확인

### 실행 방법

| 순서 | 화면에서 할 일                         | 예상 결과                                    |
| :--: | -------------------------------------- | -------------------------------------------- |
|  1   | 메인 화면에서 `강광민` 으로 검색합니다 | **0건** — "일치하는 문서를 찾지 못했습니다." |
|  2   | `홍길동` 으로 검색합니다               | **0건** — 결과 없음                          |
|  3   | `NONAME999` 로 검색합니다              | **0건** — 결과 없음                          |

**✅ 합격 기준:** 존재하지 않는 이름 검색 시 결과 0건 (엉뚱한 문서 노출 없음)
