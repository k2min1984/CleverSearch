# 폐쇄망 대국민 서비스 설명 요약 (비개발자용)

작성일: 2026-04-08

## 1. 한눈에 보기

- 현재 구조는 검색엔진(OpenSearch) + 운영 DB 분리 방식입니다.
- 이 구조 자체는 대국민 서비스에서도 널리 쓰는 표준 구조입니다.
- 다만 대국민 대규모 운영을 위해서는 이중화, 무중단 배포, 장애 자동복구 같은 운영 고도화가 추가로 필요합니다.

## 2. 업로드 방식 (현재 처리)

1. 사용자가 파일 업로드
2. 서버가 확장자/용량 검사
3. 파일 바이트로 SHA-256 해시 계산
4. 같은 해시면 중복으로 건너뜀(skipped)
5. 중복이 아니면 텍스트 추출(OCR 포함)
6. OpenSearch에 검색용 색인 저장
7. 앱 DB(indexed_documents)에 문서 메타/해시 저장

한 줄 설명:
검색은 OpenSearch, 운영 이력 관리는 DB로 역할을 나눠 처리합니다.

## 3. 검색 방식 (현재 처리)

1. 사용자가 검색어 입력
2. 검색어 정제(특수문자 제거), 오타 보정, 사전 보정 적용
3. OpenSearch에서 하이브리드 검색 실행
4. 초성 입력은 초성 전용 필드로 별도 처리
5. 결과 반환 후 검색 로그와 최근 검색어를 DB에 저장

참고:
하이브리드 검색은 키워드 일치(BM25)와 의미 유사도(KNN)를 함께 반영합니다.

## 4. 추천 검색어는 어떻게 나오나

- 복잡한 AI 예측이 아니라 운영 데이터 기반 병합 방식입니다.
- 우선순위:

1. 사용자 최근 검색어
2. 전체 인기 검색어(최근 기간 집계)

- 두 목록을 합쳐 중복 제거 후 상위 N개를 보여줍니다.

한 줄 설명:
내가 최근 찾은 것과 사람들이 많이 찾은 것을 합쳐 추천합니다.

## 5. 검색 품질 관리 안내 위치

- 검색 품질(가중치) 상세 설명은 관리자 메뉴 설명 문서로 분리했습니다.
- 참고: docs/관리자_메뉴_설명.md 의 "검색 품질 탭" 섹션

## 6. 공유용 짧은 멘트

- 업로드는 중복 검사 후 OpenSearch와 DB에 역할 분리 저장합니다.
- 검색은 키워드와 의미 유사도를 함께 사용합니다.
- 추천어는 최근 검색어와 인기 검색어를 합쳐서 제공합니다.
- 검색 품질(가중치)은 관리자 메뉴의 검색 품질 탭에서 조정합니다.
