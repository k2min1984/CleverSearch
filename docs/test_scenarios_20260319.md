## 2. 사전 준비

### 2-1. 서버 확인

1. 서버 실행 확인
2. 메인 화면(/) 접속
3. 관리자 화면(/admin) 접속 → **로그인 화면이 먼저 나오는지 확인**

### 2-2. 초기화

1. /admin 로그인 후 **초기화** 탭 이동
2. "전체 데이터 초기화" 클릭 → OpenSearch + DB(문서목록·검색로그·최근검색어) 동시 삭제 확인
3. 문서 목록 탭에서 총 0건 확인

## 3. 관리자 페이지 (/admin)

### 3-1. 로그인/로그아웃

| 단계 | 조작                             | 기대결과                                    |
| ---- | -------------------------------- | ------------------------------------------- |
| 1    | /admin 접속                      | 로그인 화면만 보임. 탭/내부 콘텐츠 안 보임  |
| 2    | admin / admin123! 입력 후 로그인 | 탭 + 콘텐츠 노출. 상단에 사용자명 표시      |
| 3    | 로그아웃 클릭                    | 로그인 화면으로 돌아감. 내부 콘텐츠 안 보임 |
| 4    | 잘못된 비밀번호로 로그인 시도    | "로그인 실패" 에러 메시지 표시              |

### 3-2. 파일 업로드

| 단계 | 조작                                 | 기대결과                                       |
| ---- | ------------------------------------ | ---------------------------------------------- |
| 1    | 파일 업로드 탭 클릭                  | 드래그 앤 드롭 영역 노출                       |
| 2    | 파일 1~6을 드래그 or 클릭으로 업로드 | 각 파일명이 표시되며 업로드 진행               |
| 3    | 업로드 완료 후 결과 확인             | 파일명 + 상태(success/skipped/fail)가 1:1 표시 |
| 4    | 문서 목록 탭에서 새로고침            | total = 6, 파일명·확장자·카테고리 확인         |
| 5    | 중복 파일(test.\*) 6개 업로드        | test.\* 파일도 success 표시(첫 업로드 시)      |
| 6    | 동일 파일 재업로드                   | skipped 상태로 표시(중복 차단)                 |

### 3-3. 문서 관리

| 2    | 특정 문서의 삭제 클릭   | 확인 팝업 → 삭제 완료 메시지 → 목록에서 사라짐 |

## 8. 합격 기준

1. 관리자 페이지 로그인/로그아웃 정상 동작 (비인증 시 콘텐츠 숨김)
2. 파일 업로드 시 파일명 + 상태 표시, 중복 차단
3. 전체 초기화 시 OpenSearch + DB 동시 정리
4. 개별 문서 삭제 가능
5. 기본 검색에서 의도 문서 상위 노출
6. 상세검색 필터(include/exclude/확장자/기간) 정상 동작
7. 자동완성/인기검색/최근검색/실패검색 정상 기록
8. 대시보드 지표 표시
9. 전 구간 실행 증적(스크린샷 또는 로그)을 남길 것
