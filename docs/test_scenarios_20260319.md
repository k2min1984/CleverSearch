## 3. 메인 검색 화면 테스트 (/)

### 3-1. 업로드/중복/색인 검증

검증 번호: 8, 9, 10, 11, 12, 13, 14

절차

1. 파일 1~6을 관리자 화면에서 업로드
2. 업로드 진행 영역(또는 토스트/로그)에서 각 파일명이 표시되는지 확인
3. test.\* 파일 6개를 연속 업로드
4. 업로드 결과 목록에서 파일명 + 상태(success/skipped/failed) 조합이 보이는지 확인
5. 관리자 문서 목록 total 확인

기대결과

- 원본 파일은 success
- 중복 파일은 skipped 또는 중복 차단
- 업로드 중/완료 피드백에 어떤 파일인지 파일명이 명확히 표시됨
- 결과 행마다 파일명과 상태가 1:1로 대응됨
- 문서 목록 total은 중복 제외 건수와 일치

### 3-2. 기본 검색 매칭/정렬

검증 번호: 24, 25, 30

절차

1. 산업기술혁신사업 검색
2. 연구개발계획서 검색
3. 운영지침, 보안 검색
4. 상위 노출 문서 제목 확인

기대결과

- 의도 문서가 상위에 노출
- 검색 total이 상식적 범위로 반환

### 3-3. 상세검색/필터/기간

검증 번호: 26, 27, 28

절차

1. 검색어 연구개발계획서 입력
2. include_keywords=울산지방청 적용
3. exclude_keywords=작성중 적용
4. file_ext=pdf 적용
5. 기간을 당일로 제한

기대결과

- include 미포함 문서 제외
- exclude 포함 문서 제외
- PDF/기간 조건이 동시에 반영

### 3-4. 이름 검색 정밀도

검증 번호: 29

절차

1. 강광민, 홍길동, 김영희 검색
2. 결과 문서 제목/본문에 이름 실재 여부 확인

기대결과

- 실제 미포함이면 total=0
- 의미유사만으로 엉뚱한 결과 노출 시 실패

### 3-5. 자동완성/인기/오타

검증 번호: 31, 34, 35

절차

1. 검색창에 연구 입력 후 자동완성 확인
2. 연구개발계획서/산업기술혁신사업/보안을 반복 검색
3. 연구개발계휙서, 산엽기술혁신사업 입력

기대결과

- 자동완성 후보 노출
- 인기검색 집계 증가
- 오타 질의에서도 오류 없이 응답

### 3-6. 최근/추천/연관검색

검증 번호: 32, 33, 36

절차

1. 동일 user_id로 검색어 5개 이상 실행
2. 최근검색 목록 확인
3. 추천검색 목록 확인
4. 연관검색(연구개발계획서) 확인
5. 최근검색 단건삭제 및 전체삭제

기대결과

- 최근검색 최신순 반영
- 추천/연관검색이 데이터 기반으로 반환
- 삭제 후 목록 즉시 반영
